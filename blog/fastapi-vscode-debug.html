1. Why the print() Habit Hurts

Every Python developer has done it: you hit a bug, you scatter a handful of print(some_var) statements across your handler, restart the server, fire a request, squint at the terminal, then do it all over again. It works — eventually — but it is slow, noisy, and teaches you nothing about the control flow you did not already suspect.

A proper interactive debugger gives you:

  • Breakpoints — pause execution at any line without touching source code
  • Variable inspection — see the full object tree, not just what you remembered to print
  • Step-through execution — watch exactly which branches your code takes
  • Call-stack navigation — jump between frames to understand how you got here
  • Expression evaluation — run arbitrary code in the paused context

FastAPI + VS Code makes all of this trivially easy once you have a launch.json in place. Here is the exact setup I use in production-like environments.

2. Prerequisites

Make sure the following are installed before proceeding:

  • VS Code with the Python extension (ms-python.python)
  • Python 3.10+ virtual environment activated
  • fastapi, uvicorn[standard] installed in that environment
  • Optional: Docker + Remote – Containers extension for the Docker section
Note All commands below assume your project root contains main.py and your virtual environment is at .venv/. Adjust paths as needed.

3. Setting Up launch.json

In VS Code, press Ctrl+Shift+P (or Cmd+Shift+P on Mac) and select Debug: Open launch.json. If no file exists yet, VS Code creates .vscode/launch.json for you. Paste the following configuration: 

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "FastAPI: Uvicorn",
      "type": "debugpy",
      "request": "launch",
      "module": "uvicorn",
      "args": [
        "main:app",
        "--reload",
        "--host", "0.0.0.0",
        "--port", "8000"
      ],
      "jinja": true,
      "justMyCode": true
    }
  ]
}

Key fields explained:

  • "module": "uvicorn" — tells debugpy to run python -m uvicorn rather than a script file, which is the correct way to launch uvicorn so it inherits the debugger's process
  • "--reload" — keeps hot-reload active; breakpoints survive file saves
  • "justMyCode": true — skips stepping into FastAPI / Starlette internals unless you explicitly want to
Warning Do not use "program": "path/to/uvicorn" — that launches the uvicorn binary directly and debugpy cannot inject itself. Always use "module".

4. Running FastAPI Under the Debugger

With launch.json saved, open your main.py and click the gutter to the left of a line number inside any route handler. A red dot appears — that is your breakpoint. 

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    price: float

@app.post("/items/")
async def create_item(item: Item):
    # ← Click the gutter here to set a breakpoint
    result = {"item_name": item.name, "item_price": item.price}
    return result

Press F5 (or click the green play button in the Run & Debug sidebar). VS Code starts uvicorn and attaches debugpy. Now send a request: 

curl -X POST http://localhost:8000/items/ \
  -H "Content-Type: application/json" \
  -d '{"name": "widget", "price": 9.99}'

VS Code pauses at your breakpoint. You can hover over item to see its full Pydantic model, use the Debug Console panel to evaluate item.price * 1.1, or step through the function with F10 (step over) and F11 (step into).

Pro tip Open the Watch panel and add expressions like len(result) or item.model_dump(). They re-evaluate every time execution pauses, giving you a live dashboard of your key values.

5. Debugging Inside Docker

If your FastAPI app runs in a Docker container (common in microservice setups), you need to expose the debugpy port and tell VS Code to attach to it remotely.

5.1 Update your Dockerfile

FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt debugpy

COPY . .

# Use debugpy to launch uvicorn — port 5678 is the debugger, 8000 is the app
CMD ["python", "-m", "debugpy",
     "--listen", "0.0.0.0:5678",
     "--wait-for-client",
     "-m", "uvicorn", "main:app",
     "--host", "0.0.0.0", "--port", "8000", "--reload"]

5.2 Expose the port in docker-compose

services:
  api:
    build: .
    ports:
      - "8000:8000"   # FastAPI
      - "5678:5678"   # debugpy
    volumes:
      - .:/app        # live reload picks up file changes

5.3 Add a remote-attach configuration

Add a second entry to your launch.json configurations array:

{
  "name": "FastAPI: Docker Attach",
  "type": "debugpy",
  "request": "attach",
  "connect": {
    "host": "localhost",
    "port": 5678
  },
  "pathMappings": [
    {
      "localRoot": "${workspaceFolder}",
      "remoteRoot": "/app"
    }
  ],
  "justMyCode": true
}

Run docker compose up in a terminal. The container starts and waits for the debugger (because of --wait-for-client). In VS Code, select FastAPI: Docker Attach from the debug dropdown and press F5. The container unblocks, uvicorn starts, and your breakpoints work exactly as they do locally.

Security reminder Never expose port 5678 in production. The --wait-for-client flag also blocks all requests until a debugger connects — use a separate docker-compose.debug.yml override file rather than modifying your production compose file.

6. Tips & Tricks

Conditional breakpoints

Right-click a breakpoint dot → Edit Breakpoint → add a condition like item.price > 100. The debugger only pauses when the condition is true — invaluable when you need to catch a specific case inside a loop or high-traffic endpoint.

Logpoints (no-stop tracing)

Right-click the gutter → Add Logpoint. Enter a message like Processing item: {item.name}, price={item.price}. VS Code logs this to the Debug Console on every pass — without pausing execution. Think of it as a structured, zero-code print() that you can toggle off in one click.

Debugging background tasks

from fastapi import BackgroundTasks

@app.post("/send-email/")
async def send_email(background_tasks: BackgroundTasks, email: str):
    background_tasks.add_task(heavy_email_sender, email)
    return {"message": "queued"}

async def heavy_email_sender(email: str):
    # Breakpoints work here too — FastAPI runs background tasks
    # in the same event loop, so debugpy stays attached

Exception breakpoints

In the Run & Debug sidebar, expand Breakpoints at the bottom. Check Raised Exceptions or Uncaught Exceptions. VS Code will pause the moment any exception is raised — before FastAPI's exception handlers swallow it — letting you inspect the exact state that caused the error.

7. Wrapping Up

The print() habit is understandable — it is fast to type and universally available. But the moment your FastAPI app grows beyond a handful of routes, or you need to debug an async race condition, or you are chasing a Pydantic validation error three stack frames deep, an interactive debugger pays for itself immediately.

The setup described here — launch.json with "module": "uvicorn" for local work and a debugpy --listen CMD for Docker — covers 95% of the debugging scenarios I encounter day to day. Set it up once, commit it to your repo, and your whole team gets it for free.

TL;DR checklist
  • Add .vscode/launch.json with "module": "uvicorn"
  • Use "justMyCode": true to stay focused on your code
  • For Docker: expose port 5678, install debugpy, use --wait-for-client
  • Try conditional breakpoints and logpoints before reaching for print()
